# Navlyn 官网架构与内容交付文档

## 1. 文档目的

本文件用于把现有“官网架构思维导图”扩展为可执行的网站交付说明，方便设计、前端、内容和商务统一理解。

适用范围：

- 官网信息架构确认
- 原型设计依据
- 内容准备清单
- 前端开发范围界定

## 2. 官网定位

Navlyn 官网不是传统企业展示站，而是同时承担以下职能：

- 品牌宣传
- 技术教育
- 行业方案销售
- 渠道合作转化
- 国际化业务承接

一句话定位建议：

`以 ARC OS 为核心，面向低空经济场景提供飞行智能体系统、硬件终端与行业解决方案。`

## 2.1 首版范围收缩建议

基于当前素材情况，首版官网建议采用“轻量发布”策略：

- 只做最核心的品牌、产品、技术和联系转化页面
- 避免一开始就拆出过多二级栏目
- 优先上线能承接商务咨询和品牌展示的页面

首版建议页面：

- Home
- Products
- ARC OS
- Industries
- About Us
- Contact / Demo

## 3. 用户类型

| 用户类型 | 核心诉求 | 推荐入口 |
| --- | --- | --- |

| 媒体 / 求职者 | 品牌背景、新闻、团队 | About Us |

## 5. 页面内容要求

## 5.1 Home 首页

必须出现的内容：

- 品牌 slogan
- 品牌定位与三大主张
- 产品与系统能力总览
- 行业应用入口
- 成功案例
- 新闻或资源预览
- 团队 / 专家 / 全球布局背书
- 明确 CTA

成功标准：

- 用户在 1 分钟内理解公司做什么
- 用户在 2 次滚动内找到感兴趣入口
- 用户在首页即可触发预约或咨询

- 参数表
- 场景标签
- 下载入口
- 咨询入口

## 7. 多语言建议

现阶段建议采用两阶段策略：

### 阶段 1

- 中文
- 英文

原因：

- 中文和英文足以覆盖当前品牌发布与国际传播主场景
- 多语言版本必须建立统一术语表，尤其是产品名和技术名词
- 在素材不足的情况下，先不要同时维护 4 种语言

## 8. 设计与开发建议

## 8.1 设计建议

- 使用 12 栏栅格，兼顾桌面与移动端
- 产品类页面强调“图 + 参数 + 场景”
- ARC OS 页面强调“图解 + 流程 + 架构”
- 控制单页信息密度，首页长度建议在 8 到 10 个模块

## 8.2 技术建议

本项目按你的要求，建议采用以下技术栈：

- `React`
- `Vite`
- `TypeScript`
- `Ant Design`
- `React Router`
- `react-helmet-async`

推荐目录结构：

```text
src/
  assets/
  components/
  pages/
  layouts/
  sections/
  router/
  data/
  hooks/
  utils/
```

功能范围建议：

- 首版优先实现静态内容页 + 预约表单
- 产品、案例、新闻先采用本地 `data` 文件维护
- 后台和 CMS 后置，避免首期复杂度过高

## 8.3 SEO 建议

在 `Vite + React SPA` 架构下，要兼顾 SEO，需要明确一个现实前提：

- 纯客户端渲染的 SEO 天然弱于 SSR/SSG

因此首版建议采用“SPA + 预渲染”的折中方案：

- 使用 `react-helmet-async` 管理每个页面的 `title` 和 `meta description`
- 对 `Home`、`Products`、`ARC OS`、`Industries`、`About Us` 进行预渲染
- 生成 `sitemap.xml`
- 配置 `robots.txt`
- 为每个页面提供稳定 URL

如果后续 SEO 权重变高，再升级到 SSR 或 SSG 框架。

首版需要优先覆盖：

- 品牌词：Navlyn、航链科技、ARC OS
- 行业词：低空经济、飞行智能体、无人机巡检、空海协同搜救
- 产品词：Commander X1、Scout S1

每个重点页面需要有：

- 单独 title
- 单独 description
- 结构化 H1/H2
- OG 图

## 9. Ant Design 页面映射建议

为了降低实现难度，建议页面尽量采用标准组件组合：

| 页面区域 | 推荐组件 |
| --- | --- |
| 顶部导航 | `Layout.Header` + `Menu` |
| 首页卡片模块 | `Card` + `Row` + `Col` |
| 行业切换 | `Tabs` |
| 产品参数 | `Descriptions` / `Table` |
| 新闻列表 | `List` |
| 常见问题 | `Collapse` |
| 联系表单 | `Form` |
| 页脚 | `Layout.Footer` |

## 10. 里程碑建议

建议分 4 步交付：

1. 信息架构确认
2. 低保真原型
3. 高保真视觉稿
4. 前端开发与内容上线

## 11. 本轮建议确认项

如果继续往下做，建议你先确认这 5 件事：

1. 首页 CTA 最终叫“预约演示”还是“联系销售”
2. 首版一级导航是否按 5 个栏目收缩
3. `ARC Engine` 是否统一命名成 `ARC OS`
4. 首发语言是否只做中英双语
5. 首页是否突出“中法团队”作为信任背书
